LangChain Architecture Guide

The Architecture

A "ReAct" Agent (Reasoning + Acting) acts as the brain. Instead of just replying to a prompt, it enters a loop: Think → Act → Observe. To build your own agent with custom capabilities, you only need three core components.

1 Creating Custom Tools

Tools are functions the LLM can trigger. The easiest way to create one is using the @tool decorator. Crucial Rule: For classic ReAct agents, ensure your tool accepts a single string argument to avoid Pydantic parsing errors.

from langchain_core.tools import tool

@tool
def say_hello(name: str) -> str:
    """Greet the user by their name.
    Input MUST be a simple string representing the name."""
    return f"Hello, {name}! I am an Agent."

# You can create ANY number of tools like this!
# Examples: database queries, API calls, manipulating files...

The docstring (the """Text""" inside the function) is incredibly important. The LLM reads this text to figure out when to use the tool and what to pass into it.

2 Initializing the LLM

The LLM powers the reasoning. In this project, we are using Groq's lightning-fast models. You instantiate it and pass your API key (usually automatically loaded via `.env`).

from langchain_groq import ChatGroq

# Ensure GROQ_API_KEY is defined in your environments!
llm = ChatGroq(
    model="llama-3.3-70b-versatile", 
    temperature=0  # 0 keeps the agent's logic deterministic
)

3 The ReAct Loop

Finally, you bind the LLM and the Tools together into an Agent Executor. We use the `ZERO_SHOT_REACT_DESCRIPTION` pattern, which automatically formats the prompt into the Think/Act/Observe cycle.

from langchain_classic.agents import initialize_agent, AgentType

# 1. Gather tools into a list
tools = [say_hello, add_numbers, fetch_weather]

# 2. Bind them together
agent_executor = initialize_agent(
    tools=tools,
    llm=llm,
    agent=AgentType.ZERO_SHOT_REACT_DESCRIPTION,
    verbose=True,  # Prints the Thought Process to the terminal!
    handle_parsing_errors=True
)

# 3. Ask it a question!
response = agent_executor.invoke({"input": "Say hello to Harsh"})
print(response["output"])

4 Demystifying the Prompt Variables

If you look at react_loop.py, you'll see a massive ReAct prompt template. Have you wondered how the LLM perfectly tracks all the tools and its previous thoughts? This is handled by four magic variables injected into the prompt during generation:

  • {tools}: LangChain dynamically takes your @tool descriptions and injects them here so the LLM knows exactly what each tool takes as input.
  • {tool_names}: A simple comma-separated list of the names of the tools (e.g. add, get_weather) telling the LLM the exact string commands it is allowed to output.
  • {input}: The actual question you typed into the terminal.
  • {agent_scratchpad}: This is the brain's short-term memory! Every time the agent takes an Action and receives an Observation, LangChain appends that interaction onto this scratchpad. On the next loop, the LLM reads the scratchpad and knows what it has already tried so it doesn't get stuck in an infinite loop!

Summary: Sky is the Limit 🚀

To add a new capability to your bot, all you need to do is write a standard Python function, decorate it with @tool, provide a clear docstring, and append it to your tools = [...] list. The LLM will automatically learn how to use it contextually based on your docstring!
