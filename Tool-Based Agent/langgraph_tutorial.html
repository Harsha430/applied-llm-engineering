LangGraph Revolution

Migrating from AgentExecutor

Why LangGraph? The Problem it Solves

In our current project, we use the classic AgentExecutor. While easy to set up, it acts like a "black box" engine. It forces the LLM to output a specific text format (Thought/Action/Observation) via a gigantic prompt template (`{agent_scratchpad}`).

The Core Issues with Classic ReAct:

  • No Absolute Control: If the agent gets stuck in an infinite loop calling the same tool over and over, you can't easily intervene.
  • Prompt Hacking: It relies entirely on string-parsing ("Thought:", "Action:") which frequently breaks (giving Parsing Errors) if the LLM slightly deviates from the text format.
  • Brittle Memory: The `{agent_scratchpad}` grows exponentially, eventually overflowing the context window on long tasks.

LangGraph solves this entirely by moving memory to a concrete variable (a State) and making the reasoning loop a literal flowchart (a Graph) with explicit nodes and edges.

What Can We Do With LangGraph?

By mapping out execution as a stateful graph, LangGraph unlocks insane, enterprise-level capabilities that are practically impossible in standard ReAct loops:

  • Human-in-the-Loop (HITL): You can pause execution right before a risky tool (like "Delete Database_Record") runs, ask the human for permission via the UI, and then resume the graph exactly where it left off!
  • Multi-Agent Orquestration: You can create complex hierarchies where a "Researcher Agent" builds a report and passes the raw state data to a "Reviewer Agent" who grades it.
  • Time Travel: Because the graph saves its `State` at every single node safely into a database (via Checkpointers), you can rewind a conversation, change what happened at step 3, and branch off a new reality.

How to Migrate Our Project

Here is what the exact equivalent of our `react_loop.py` looks like flawlessly rewritten in LangGraph. Notice there are zero giant string prompt templates. It relies natively on OpenAI/Groq's modern Tool-Calling JSON APIs.

from typing import Annotated
from typing_extensions import TypedDict
from langgraph.graph import StateGraph, START, END
from langgraph.graph.message import add_messages
from langgraph.prebuilt import ToolNode, tools_condition
from langchain_groq import ChatGroq
from tools.calculator import add, sub, mul, div
from tools.search import search_web
from tools.api_tool import get_weather

# 1. Define the Global State (just a list of messages being passed around)
class State(TypedDict):
    messages: Annotated[list, add_messages]

# 2. Setup LLM and Bind Tools Natively
tools = [add, sub, mul, div, search_web, get_weather]
llm = ChatGroq(model="llama-3.3-70b-versatile", temperature=0)
llm_with_tools = llm.bind_tools(tools)

# 3. Create the Agent Node Function
def agent_node(state: State):
    # The LLM reads the history, decides to chat OR output a tool_call!
    response = llm_with_tools.invoke(state["messages"])
    return {"messages": [response]}

# 4. Build the Network Graph
graph_builder = StateGraph(State)

# Add our Nodes (Actors)
graph_builder.add_node("agent", agent_node)
graph_builder.add_node("tools", ToolNode(tools)) # Auto-executes any tools the LLM called

# Draw the Map (Edges)
graph_builder.add_edge(START, "agent")

# Conditional Routing: If LLM called a tool => go to "tools" node. Else => STOP.
graph_builder.add_conditional_edges("agent", tools_condition)
graph_builder.add_edge("tools", "agent") # After tool runs, loop back to agent

# Compile the app!
app = graph_builder.compile()

The Massive Advantage

In the migration code above, the LLM isn't "hallucinating" a text string like `Action: add`. Instead, the `bind_tools()` function translates your Python tools into strict JSON schemas directly inside the Groq model's neural layers natively. The LangGraph `ToolNode` automatically executes the function flawlessly without parsing errors, bundles the result into an `Observation` message, and loops back into the `agent_node`!

This removes 100% of the Pydantic type validation string issues we got during the ReAct logic, and gives you absolute granular control over the step-by-step logic map.
